Nmap - Network mapper for port scanning and service enumeration.

Target Specification

  • Scan a single IP: nmap 192.168.1.1
  • Scan specific IPs: nmap 192.168.1.1 192.168.2.1
  • Scan a range: nmap 192.168.1.1-254
  • Scan a domain: nmap scanme.nmap.org
  • Scan using CIDR: nmap 192.168.1.0/24
  • Scan targets from a file: nmap -iL targets.txt
  • Scan random hosts: nmap -iR 100
  • Exclude listed hosts: nmap -exclude 192.168.1.1

Nmap Scan Techniques

  • TCP SYN scan (Default): nmap 192.168.1.1 -sS
  • TCP connect scan: nmap 192.168.1.1 -sT
  • UDP port scan: nmap 192.168.1.1 -sU
  • TCP ACK port scan: nmap 192.168.1.1 -sA
  • TCP Window port scan: nmap 192.168.1.1 -sW
  • TCP Maimon port scan: nmap 192.168.1.1 -sM

Host Discovery

  • No Scan (List targets only): nmap 192.168.1.1-3 -sL
  • Disable port scanning (Host discovery only): nmap 192.168.1.1/24 -sn
  • Disable host discovery (Port scan only): nmap 192.168.1.1-5 -Pn
  • TCP SYN discovery on port x: nmap 192.168.1.1-5 -PS22-25,80
  • TCP ACK discovery on port x: nmap 192.168.1.1-5 -PA22-25,80
  • UDP discovery on port x: nmap 192.168.1.1-5 -PU53
  • ARP discovery on local network: nmap 192.168.1.1-1/24 -PR
  • Never do DNS resolution: nmap 192.168.1.1 -n

Port Specification

  • Port scan for port x: nmap 192.168.1.1 -p 21
  • Port range: nmap 192.168.1.1 -p 21-100
  • Scan multiple TCP/UDP ports: nmap 192.168.1.1 -p U:53,T:21-25,80
  • Scan all ports: nmap 192.168.1.1 -p-
  • Port scan from service name: nmap 192.168.1.1 -p http,https
  • Fast port scan (100 ports): nmap 192.168.1.1 -F
  • Scan top x ports: nmap 192.168.1.1 -top-ports 2000
  • Leaving off initial port in range: nmap 192.168.1.1 -p-65535
  • Leaving off end port in range: nmap 192.168.1.1 -p0-

Service and Version Detection

  • Service version detection: nmap 192.168.1.1 -sV
  • Set version intensity (0-9): nmap 192.168.1.1 -sV -version-intensity 8
  • Light mode (Faster): nmap 192.168.1.1 -sV -version-light
  • Intensity level 9 (Slower): nmap 192.168.1.1 -sV -version-all
  • Enable OS detection, version detection, script scanning, and traceroute: nmap 192.168.1.1 -A

OS Detection

  • Remote OS detection: nmap 192.168.1.1 -O
  • OS detection with limited open/closed ports: nmap 192.168.1.1 -O -osscan-limit
  • Aggressive OS guessing: nmap 192.168.1.1 -O -osscan-guess
  • Set maximum OS detection tries: nmap 192.168.1.1 -O -max-os-tries 1
  • Enable OS detection, version detection, script scanning, and traceroute: nmap 192.168.1.1 -A

Timing and Performance

  • Paranoid (IDS evasion): nmap 192.168.1.1 -T0
  • Sneaky (IDS evasion): nmap 192.168.1.1 -T1
  • Polite (less bandwidth/resource usage): nmap 192.168.1.1 -T2
  • Normal (default speed): nmap 192.168.1.1 -T3
  • Aggressive (fast/reliable network): nmap 192.168.1.1 -T4
  • Insane (extraordinarily fast network): nmap 192.168.1.1 -T5

Timing and Performance Switches

  • Timeout after x: nmap 192.168.1.1 -host-timeout 1s
  • Specify RTT time: nmap 192.168.1.1 -min-rtt-timeout 1s
  • Parallel host scan group sizes: nmap 192.168.1.1 -scan-delay 1s

    • NSE Scripts

    • List all available scripts: nmap --script-help=
    • Run a specific script: nmap --script script_name
    • Run multiple scripts: nmap --script script1,script2
    • Use a script category: nmap --script category

    Firewall/IDS Evasion Techniques

    • Fragment packets: nmap -f
    • Use a decoy: nmap -D RND:10
    • Use a spoofed IP address: nmap -S 192.168.1.100
    • Randomize target order: nmap --randomize-hosts
    • Change source port: nmap --source-port 53
    • Idle scan (stealth scan): nmap -sI 192.168.1.100

SQLMAP - Automatic SQL injection and database takeover tool.

Basic Arguments for SQLmap

  • Generic:
    • -u "<URL>"
    • -p "<PARAM TO TEST>"
    • --user-agent=SQLMAP
    • --random-agent
    • --threads=10
    • --risk=3 (MAX)
    • --level=5 (MAX)
    • --dbms="<KNOWN DB TECH>"
    • --os="<OS>"
    • --technique="UB" (Use only techniques UNION and BLIND in that order)
    • --batch (Non-interactive mode)
    • --auth-type="<AUTH>" (HTTP authentication type: Basic, Digest, NTLM or PKI)
    • --auth-cred="<AUTH>" (HTTP authentication credentials: name:password)
    • --proxy=http://127.0.0.1:8080
    • --union-char "GsFRts2" (Helps SQLmap identify union SQLi techniques with a weird union char)

Retrieve Information

  • Internal:
    • --current-user (Get current user)
    • --is-dba (Check if current user is Admin)
    • --hostname (Get hostname)
    • --users (Get usernames of DB)
    • --passwords (Get passwords of users in DB)
    • --privileges (Get privileges)
  • DB Data:
    • --all (Retrieve everything)
    • --dump (Dump DBMS database table entries)
    • --dbs (Names of the available databases)
    • --tables (Tables of a database -D <DB NAME>)
    • --columns (Columns of a table -D <DB NAME> -T <TABLE NAME>)
    • -D <DB NAME> -T <TABLE NAME> -C <COLUMN NAME> (Dump column)

Injection Place

  • From Burp/ZAP capture:

    • Capture the request and create a req.txt file:

    sqlmap -r req.txt --current-user
  • GET Request Injection:
    • sqlmap -u "http://example.com/?id=1" -p id
    sqlmap -u "http://example.com/?id=*" -p id
  • POST Request Injection:
    • sqlmap -u "http://example.com" --data "username=*&password=*"
  • Injections in Headers and other HTTP Methods:
    • sqlmap -u "http://example.com" --cookie "mycookies=*"
    sqlmap -u "http://example.com" --headers="x-forwarded-for:127.0.0.1*"
    sqlmap -u "http://example.com" --headers="referer:*"
    sqlmap --method=PUT -u "http://example.com" --headers="referer:*"
  • Indicate string when injection is successful:
    • --string="string_showed_when_TRUE"

Eval

Sqlmap allows the use of -e or --eval to process each payload before sending it with some python oneliner. This makes processing custom ways very easy and fast before sending the payload. Example:

sqlmap http://1.1.1.1/sqli --eval "from flask_unsign import session as s; session = s.sign({'uid': session}, secret='SecretExfilratedFromTheMachine')" --cookie="session=*" --dump

ffuf - Fuzzing tool for web directories and files.

Typical commands & Usage

Directory discovery: ffuf -w /path/to/wordlist -u https://target/FUZZ
Adding classical headers (for some WAF bypass): ffuf -c -w "/opt/host/main.txt:FILE" -H "X-Originating-IP: 127.0.0.1, X-Forwarded-For: 127.0.0.1, X-Remote-IP: 127.0.0.1, X-Remote-Addr: 127.0.0.1, X-Client-IP: 127.0.0.1" -fs 5682,0 -u https://target/FUZZ
Match all responses but filter out those with content-size 42: ffuf -w wordlist.txt -u https://example.org/FUZZ -mc all -fs 42 -c -v
Fuzz Host-header, match HTTP 200 responses: ffuf -w hosts.txt -u https://example.org/ -H "Host: FUZZ" -mc 200
Virtual host discovery (without DNS records): ffuf -w /path/to/vhost/wordlist -u https://target -H "Host: FUZZ" -fs 4242
Playing with threads and wait: ffuf -u https://target/FUZZ -w /path/to/wordlist.txt -c -p 0.1 -t 10
GET param fuzzing, filtering for invalid response size (or other filters): ffuf -w /path/to/paramnames.txt -u https://target/script.php?FUZZ=test_value -fs 4242
GET parameter fuzzing if the param is known (fuzzing values) and filtering 401 responses: ffuf -w /path/to/values.txt -u https://target/script.php?valid_name=FUZZ -fc 401
POST parameter fuzzing: ffuf -w /path/to/postdata.txt -X POST -d "username=admin&password=FUZZ" -u https://target/login.php -fc 401
Fuzz POST JSON data, matching all responses not containing "error": ffuf -w entries.txt -u https://example.org/ -X POST -H "Content-Type: application/json" -d '{"name": "FUZZ", "anotherkey": "anothervalue"}' -fr "error"

John the Ripper - Password hash cracker.

Cracking Modes

Dictionary attack: ./john --wordlist=password.lst hashFile
Dictionary attack using default or specific rules: ./john --wordlist=password.lst --rules=rulename hashFile

Incremental mode: ./john --incremental hashFile
Loopback attack (passwords are taken from the potfile): ./john --loopback hashFile
Mask brute-force attack: ./john --mask=?1?1?1?1?1?1 --1=[A-Z] hashFile --min-len=8
Dictionary attack using masks: ./john --wordlist=password.lst -mask='?l?l?w?l' hashFile

MISC & Tricks

Show hidden options: ./john --list=hidden-options
Using session and restoring them: ./john hashes --session=name
./john --restore=name
Show the potfile: ./john hashes --pot=potFile --show
Search if root/uid0 have been cracked: john --show --users=0 mypasswdFile
john --show --users=root mypasswdFile
List OpenCL devices and get their id: ./john --list=opencl-devices
List format supported by OpenCL: ./john --list=formats --format=opencl
Using multiple GPUs: ./john hashes --format:openclformat --wordlist:wordlist --rules:rules --dev=0,1 --fork=2
Using multiple CPUs (e.g., 4 cores): ./john hashes --wordlist:wordlist --rules:rules --dev=2 --fork=4

Wordlists & Incremental

Sort a wordlist for the wordlist mode: tr A-Z a-z < SOURCE | sort -u > TARGET
Use a potfile to generate a new wordlist: cut -d ':' -f 2 john.pot | sort -u > pot.dic
Generate candidate passwords for slow hashes: ./john --wordlist=password.lst --stdout --rules:Jumbo | ./unique -mem=25 wordlist.uniq
Incremental modes: --incremental:Lower (26 char)
--incremental:Alpha (52 char)
--incremental:Digits (10 char)
--incremental:Alnum (62 char)
Create a new charset: ./john --make-charset=charset.chr
Use a specific charset: ./john --incremental:charset hashFile

Rules

Predefined rules: --rules:Single, --rules:Wordlist, --rules:Extra, --rules:Jumbo, --rules:KoreLogic, --rules:All
Create a new rule in John.conf: 
[List.Rules:Tryout]
l
u
...

Sample Rule Descriptions

  • l: Convert to lowercase
  • u: Convert to uppercase
  • c: Capitalize
  • l r: Lowercase the word and reverse it
  • l Az"2015": Lowercase the word and append "2015"
  • d: Duplicate
  • l A0"2015": Lowercase the word and append "2015" at the beginning
  • A0"#"Az"#": Add "#" at the beginning and end of the word
  • C: Lowercase the first char and uppercase the rest
  • r: Reverse the word
